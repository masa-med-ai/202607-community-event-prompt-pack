# レベル5 — PPT スキルでスライド生成（デザインも対話で決定）

**ねらい**: 標準の `pptx` スキルを使い、テーマ固定のスライドを生成する。デザインの決め方（おまかせ／相談）も体験する。

**スライドテーマ（固定）**: 「**LLM を用いた臨床推論の前向き研究**」
（例: 救急外来の鑑別診断で LLM 支援が診断精度・時間に与える効果を検証する前向き研究、という想定でデモ構成を作る）

**所要時間**: 約15分。

---

## Claude への指示

### Step 0. 解説

1行で伝える: 「`pptx` スキルを使うと、構成案からそのまま編集可能な PowerPoint を生成できます。今日はテーマを固定して、デザインの決め方を体験します」。

### Step 1. デザインの決め方を選ばせる（AskUserQuestion）

次の2択を提示する:

- **A. Claude におまかせ** — テーマに合う配色・フォントで自動デザインする。
- **B. 相談して決める** — 下記の項目を選んでもらう。

### Step 2-B. 「相談して決める」を選んだ場合

`AskUserQuestion` は1回で最大4問のため、**2バッチに分けて**まとめて聞く（基本4項目 → 方向性4項目）。各項目は2〜4択で提示し、迷ったときの推奨を1つ目に置く。

**バッチ1：基本の見た目**

1. **配色**: 明るい（白背景・濃い文字） / 暗い（ダーク背景）
2. **フォント**: ゴシック系（メイリオ） / 明朝系
3. **文字サイズ・情報密度**: 各スライド最大7行まで（余白多め・発表向き） / 行数制限なし（情報量重視）
4. **コーポレートカラー**: 指定なし（テーマに合わせてClaudeが選ぶ） / 指定あり（色を聞く。例: 昭和大学カラー等）

**バッチ2：デザインの方向性**

5. **用途・場面**: 学会発表（口頭前提・要点だけ） / 院内勉強会・抄読会（やや詳しめ） / 一般向け講演（平易・大きめ文字） / 配布資料（読んで分かる情報量）
6. **テイスト（雰囲気）**: アカデミック堅実（信頼感重視） / ミニマル＆余白重視 / モダン＆先進的 / 親しみやすい・やわらかい
7. **ビジュアル量**: アイコン・図表を積極的に使う / バランスよく / テキスト中心でシンプルに
8. **サンドイッチ構成**: タイトルと結論だけ濃色背景にする（締まりが出る） / 全ページ同系の明るさで統一

聞き取った内容は、生成前に1〜2行で要約して参加者に確認する（例:「学会発表向け・アカデミック堅実・図表多め・サンドイッチ構成で作ります」）。矛盾する選択（例: 明るい配色＋全ページ濃色）があれば指摘して調整する。

> 時間が限られる場合は、バッチ1の4項目だけ聞き、バッチ2は「方向性はおまかせでいい？」と一括確認してもよい。

### Step 2-A. 「おまかせ」の場合

テーマに合う落ち着いた医学プレゼン向けデザイン（明るい背景・ゴシック・1スライド7行以内・アクセントカラー1色）で進める旨を伝える。

### Step 3. 構成案を提示

スライドを生成する前に、構成案（アウトライン）を提示して合意を取る。デフォルト構成例:

1. タイトル（研究名・演者）
2. 背景（臨床推論における LLM 活用の現状と課題）
3. リサーチクエスチョン / 目的（PICO）
4. 方法（前向き研究デザイン・対象・介入・主要評価項目）
5. 主要アウトカムと統計
6. 想定される結果・図表のイメージ
7. 強みと限界
8. 結論・臨床的含意

参加者が修正を希望すれば反映する。

### Step 4. 生成

`pptx` スキルを読み込み、決めたデザインと構成で `.pptx` を生成する。Step 2 の選択を次のように反映する:

- バッチ1（配色・フォント・情報密度・コーポレートカラー）はそのまま見た目に反映。
- バッチ2の方向性も具体化する:
  - **用途**: 学会発表→要点のみ大きく／配布資料→説明文を厚めに、など密度を調整。
  - **テイスト**: アカデミック堅実→落ち着いた配色と整然としたレイアウト／モダン→コントラスト強め・大きな数字。
  - **ビジュアル量**: 「積極的」ならアイコン・グラフ・カードを多用、「シンプル」なら装飾を抑える。
  - **サンドイッチ構成**: 採用ならタイトル・結論を濃色背景にし、中身は明るく。
- 生成後は `pptx` スキルの QA 手順（画像化して目視確認）でテキストのはみ出し等をチェックしてから共有する。

### Step 5. 共有

生成した `.pptx` を `present_files` で開けるようにし、「ここから自由に編集できます」と伝える。デザインの微調整希望があれば1〜2点まで対応する。

### Step 6. 気に入ったら Skill 化を提案

仕上がりを見せたあと、必ず次のように尋ねる:

> **「このデザイン（配色・フォント・構成・レイアウト）が気に入ったら、Skill にして次回から同じ型で作れるようにできます。Skill 化しますか？」**

`AskUserQuestion` で「Skill にする／今回はしない」を選んでもらう。

**「Skill にする」を選んだ場合:**
1. 今回確定したデザインルールを言語化する — 配色（背景・本文・アクセントの各カラーコード）、フォント、1スライドの行数上限、サンドイッチの有無、カード/アイコンのレイアウト規則、標準の構成（セクションの並び）。
2. それらを手順書（`SKILL.md`）にまとめ、`.skill` パッケージとして書き出す。`skill-creator` スキルがあれば活用する。
3. `present_files` で `.skill` を渡し、「カードの『Save skill』ボタン、または設定 → Capabilities からインストールできます」と案内する。
4. 補足: このセッションから既存 Skill を直接書き換えることはできないため、提供するのは「インストール用パッケージ」である旨を一言添える。

**「今回はしない」を選んだ場合:**
- 軽量な代替として「このフォルダの `CLAUDE.md` に “スライドは既定でこの配色・フォントで” と1行書いておけば、次回から既定値として効きます」と案内してもよい。

完了後「次はどのレベルを体験しますか？」と尋ね、`00_START` の Phase 1 に戻る。
