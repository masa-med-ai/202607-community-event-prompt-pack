# ファシリテーター用 README — Cowork / Claude ワークショップ

## 進め方

1. 参加者に、このフォルダ一式を自分の Cowork 作業フォルダへ配置してもらう。
2. 参加者が Claude に次のように伝える:
   > `00_START_ここから.md` を読んで、ワークショップを始めて。
3. Claude が **環境チェック → レベル選択 → 体験** を自走する。
4. 1レベル終わるごとに `_修了記録.md` にスタンプ（1行）が追記され、「別レベルも体験するか」を尋ねるので、時間に応じて複数体験できる。レベル選択メニューには所要時間の目安が表示される。
5. つまずいたら `FAQ_つまずいたら.md`、CLAUDE.md や Memory のしくみの質問には `付録_CLAUDEmdとMemoryのしくみ.md` を使って案内される。

## 事前準備（ファシリテーター側）

- **MCP 接続の確認**: PubMed MCP（`pubmed_mcp_server2`）、Arxiv MCP、**Gmail MCP・Google カレンダー MCP**（レベル8）。※ **Notion MCP は今回のハンズオンでは使用しない**（確認・導入不要）。
  - 本パック作成時点では **PubMed は接続確認済み、Arxiv は未接続**。レベル4を使うなら PubMed は必須。
  - **レベル8は Gmail と Google カレンダーのコネクタが必須**（Claude 標準コネクタ。設定 → コネクタ から追加し OAuth 認証。`Gog-CLI` ではなくこちらを使う）。接続は当日その場でも可能。
  - **レベル9（定期実行タスク）は PC の電源 ON ＋ Cowork 起動中でないと実行されない**。①予定確認はカレンダー MCP、②論文・ニュースは PubMed MCP ＋ Web検索を使う。体験では「今すぐ試走」で出力を見せてからスケジュール登録する。
- **レベル10（Desktop版 Claude Code）**: 参加者は**自分の Claude Desktop アプリで「Code」タブを操作**する（Cowork は手順表示・伴走のみ）。ターミナルの別ウィンドウは不要で、**同じアプリのタブを切り替えるだけ**。**Claude Desktop にサインイン済み・Code タブが使えること（Pro/Max/Team/Enterprise）が前提**（未導入者は事前セットアップ推奨。手順書末尾に付録あり。Windows は初回に Git for Windows が必要）。実ファイルを編集できるため、最初は **Ask permissions（確認）モード**のまま・差分を読んでから承認・秘密情報は渡さない、を周知する。CLI 版を使いたい人向けの案内は付録に残してある。
- **レベル11（機械学習・画像分類）**: 本編は Cowork のサンドボックスで**オフライン動作する公開教材データ（scikit-learn の手書き数字）**を使う（ダウンロード不要）。**医療画像版（MedMNIST）はサンドボックスのネット制限でダウンロード不可のため、デスクトップ版 Code / CLI で実行**する発展扱い。教育デモであり**診断・医療機器ではない**こと、実患者画像は使わないことを必ず周知する。
- **レベル12（自動化の3層）**: レベル9の発展回。**ハンズオンは Cowork 内で完結する Schedule（定期タスク）を1つ作る**だけ（レベル9と同じく PC ON＋Cowork起動が実行条件）。`/loop`（CLI）と Routines（クラウド）は**比較表とコマンドの紹介**に留め、その場で動かさない。Routines はクラウドで自律実行・別枠の使用量上限（24h で5回）がある点だけ触れる。`/loop` の自動失効期間など細部は情報源で記述が分かれるため断定しない。
- **Skill の確認**: `pptx`（レベル6）、`pubmed-systematic-review`（レベル5）、`docx`・`xlsx`（レベル3）。**レベル5の Skill は本フォルダに同梱済み**（`Agent-Skills-main/pubmed-systematic-review.skill`）。未導入でも当日その場の `.skill` から導入、または展開して中の `SKILL.md` を読んで手動進行できる。
- **レベル3（Word テンプレート差し込み）**: 同梱の `理由書フォーマット.docx`（罫線・網掛け付きの様式）を、体裁を崩さず対話で埋める。**原本は上書きせず必ず別名保存**（例: `理由書_記入済み.docx`）。差し込みは docx スキルで行い、空欄・空セルにテキストを入れるだけに徹する（このテンプレは unpack→pack の検証でまれにエラーになるため、`word/document.xml` を直接置換して zip を書き戻す方式が安全）。仕上げに PDF 変換、発展で Excel（xlsx）への差し込みを見せる。機微情報・患者情報は書き込まない。
- **配布物**: 参加者にこのフォルダを共有（Dropbox 共有リンク / USB / 事前ダウンロード等）。

## 安全・運用上の注意

- 患者個人情報は入力・生成しない（レベル1のCSVはダミー、レベル7のChrome拡張は完全ローカル動作・サーバ送信なしで設計済み）。
- PubMed 検索は MeSH で検索式を立て、`count` で妥当性を検証してから本検索する作法を全モジュールで徹底。
- 文献を引用する際は第一著者・雑誌名・発刊年・PubMed リンクを必ず付す。
- レベル3では原本テンプレート（`理由書フォーマット.docx`）を上書きせず別名保存し、患者情報・機微情報は書き込まない。
- レベル7で作る Chrome 拡張は完全ローカル動作（外部送信なし）。読み込みは「パッケージ化されていない拡張機能」として各自のブラウザに。
- レベル8では各自の**実メール・実カレンダー**を扱う。カレンダー登録は**必ず確認してから**行い（自動即登録はしない）、機微情報・患者情報は予定のタイトル/説明に転記しない。デモ環境では個人アカウントの取り扱いに留意する。

## カスタマイズのヒント

- 参加者層が非医療系中心なら、レベル1のダミーファイルや レベル7の拡張機能の題材を一般向けに差し替えると良い（各モジュール冒頭の「ねらい」は流用可）。
- 時間が短い場合は「レベル1＋もう1つ」に絞ると1人あたり20〜30分で回せる。
