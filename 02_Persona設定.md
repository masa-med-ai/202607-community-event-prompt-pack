# レベル2 — Persona（あなた専用設定）を対話で作る

**ねらい**: Claude に「あなたが誰で、どんな文章を好むか」を一度教えておくと、以降の全作業がぐっと自分仕様になる。その設定ファイルを対話形式で作る。

**所要時間**: 約10分。

---

## Claude への指示

### Step 0. 解説

まず1行で伝える: 「Persona とは、あなたの職業・好み・文体を書いた設定メモです。これを置いておくと、Claude が毎回あなた仕様で動きます」。

### Step 1. 対話で項目を埋める（AskUserQuestion を使う）

`AskUserQuestion` で、できるだけ**選択肢方式**で聞く。自由記述は最小限にする。複数の質問はまとめて提示してよい。

聞く項目:

1. **職業・役割**（例: 臨床医 / 研究者 / 教育者 / 開業医 / その他）
2. **よく作る文書**（複数選択可: 学会抄録 / 論文 / 患者説明文書 / 院内資料 / 講演スライド / SNS発信 / メール返信）
3. **日本語の文体**（である調（論文寄り） / ですます調（説明・患者向け） / カジュアル（SNS寄り））
4. **英語を使うか**（よく使う / たまに / ほぼ使わない）
5. **トーンの好み**（簡潔・端的 / 丁寧で網羅的 / 温かみ重視）
6. **箇条書きの好み**（積極的に使う / 最小限にして散文中心）
7. **長さの好み**（短く要点だけ / 標準 / 詳しく）
8. **PubMed検索結果の引用表示**（する / しない） — 次のようにおすすめを添えて聞く: 「PubMed検索を行い回答に反映させた場合は、**第一著者・雑誌名・発刊年を記載し、これにハイパーリンクをつける**のがおすすめですが、そのように表示しますか？」。「する」を選んだ場合は、Persona に次のルールを書き込む — `PubMed検索を行い回答に反映させた場合は1st author, 雑誌名, 発刊年を記載し、これにハイパーリンクをつけること`。

必要に応じて専門領域（診療科など）も1問追加してよい。

### Step 2. 保存先を選ばせる（AskUserQuestion）

次の2択を提示する:

- **A. `Persona.md` を作る（推奨）** — 設定を独立ファイルにまとめる。後から見直し・編集しやすい。
- **B. `CLAUDE.md` に直接書く** — 1ファイルに集約したい人向け。

### Step 3. ファイル生成

**A を選んだ場合:**
1. 作業フォルダに `Persona.md` を作成し、Step 1 の回答を整形して書き込む（見出し: 基本情報 / 文書の種類 / 文体・トーン / その他の好み）。
2. 同フォルダの `CLAUDE.md` を確認する。
   - 既にある場合: 先頭付近に次の1行を追記する — `> このフォルダで作業する際は、同階層の Persona.md がある場合は必ず参照すること。`
   - 無い場合: `CLAUDE.md` を新規作成し、その参照行のみ書く。
3. 「次回からこのフォルダで作業すると、自動でこの Persona が効きます」と伝える。

**B を選んだ場合:**
1. `CLAUDE.md` に「## Persona（私について）」セクションを追記する形で回答を書き込む。
2. 既存の `CLAUDE.md` がある場合は、既存内容を壊さず末尾に追記する。

### Step 4. 動作確認（任意）

「試しに、この Persona で短いサンプルを書いてみますか？」と提案する。希望があれば、設定した文体・トーンで2〜3文の例（例: 学会抄録の冒頭、または患者説明文の一文）を生成して効果を実感してもらう。

### Step 5. 過去の文書から文体のくせを抽出する（任意）

ここで一歩進んだ提案をする: 「これまでにご自身で書いた文書を何点か読み込ませると、語尾や言い回しなど“あなたらしさ”を自動で抽出して Persona に足せます。やってみますか？」。

希望した場合の進め方:

1. **文書を集めてもらう** — 「文体の参考にしたい文書（Word / PDF / テキスト）を2〜4点、このフォルダに置くか、この会話に添付してください」と伝える。論文・抄録・患者説明文・メールなど、種類が混ざっていても構わない（むしろ用途別の違いが見えてよい）。Cowork では複数の `.docx` をまとめて読み込み・解析できる。
2. **テキストを抽出する** — 各ファイルから本文を取り出す（`.docx`/`.pdf` は中身を読み取る）。表紙・定型ヘッダーなど文体と無関係な部分は除く。
3. **くせを軽く抽出する** — 次の観点で主要な傾向だけを箇条書きにまとめる:
   - 語尾（である調 / ですます調、断定の強さ）
   - 一文の長さ（短文中心 / 長め）
   - 頻出する語・言い回し・接続表現
   - 箇条書きの多寡、敬語・クッション言葉の度合い
   - 数字・専門用語の使い方
4. **確認してから追記する** — 抽出した傾向をユーザーに提示し、「どれを Persona に採り入れますか？」と選んでもらう。採用分だけを `Persona.md`（B を選んでいれば `CLAUDE.md`）の「文体・トーン」に追記する。意図しないくせを書き込まないよう、必ず確認を挟む。

> メモ: 読み込ませた文書はあくまで文体抽出のための参考であり、内容を保存・転載するものではないことを一言添えると安心感が増す。

完了後「次はどのレベルを体験しますか？」と尋ね、`00_START` の Phase 1 に戻る。
