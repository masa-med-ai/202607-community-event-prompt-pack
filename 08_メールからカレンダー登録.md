# レベル8 — 未読メールからカレンダー登録（Gmail × カレンダー MCP → スキル化）

**ねらい**: Claude 標準の **Gmail MCP** と **Google カレンダー MCP** を接続し、未読メールから予定（会議・アポイント・学会など）を拾って、**確認のうえ**カレンダーへ登録する一連を体験する。うまく回ったら、その手順を **Skill 化**して、毎日くり返せる自動化につなげる。

**ポイント**: 今回は `Gog-CLI` ではなく、**Claude 標準の Gmail / Google カレンダー コネクタ（MCP）**を使う。設定→コネクタから接続するだけで、Claude が直接メールを読み・予定を作れるようになる。

**所要時間**: 約20分。

---

## Claude への指示

> 進行の前提: 患者個人情報や機微な内容はメール本文に含まれていても**カレンダーのタイトル・説明に転記しない**。予定の登録は**必ず参加者の確認を取ってから**行う（自動で勝手に登録しない）。

### Step 0. 解説（まず MCP のしくみを図解で示す）

いきなり手を動かす前に、**まず最初にインフォグラフィックを表示**して、「そもそも MCP とは何か」を腹落ちさせる。

1. **参考図解 `images/MCPとは_USB-C解説.html` をチャットに提示**する（`present_files` で開けるようにする）。これは1枚で MCP のしくみをまとめたインタラクティブな図解で、右側のサービス（Gmail・カレンダー等）をクリックすると、その MCP で何ができるかが表示される。

2. この図を見せながら、専門用語を避けて次の3点を噛み砕いて説明する。

   **① MCP は「共通の差込口」**
   「MCP（Model Context Protocol）は、**Claude と外部サービスをつなぐ共通の差込口**です。スマホやノートPCの **USB-C** のように、規格が共通だから、いろいろなサービスをそのまま“差し込める”——それと同じで、MCP に対応したサービス（Gmail・カレンダー・PubMed など）は Claude にそのまま差し込めて、Claude が自分でそのサービスを操作できるようになります」。要点は「Claude が“会話だけ”の存在から、“外部の道具を実際に使える”存在になる」こと。

   **② MCP は「コネクタ」とも呼ばれる**
   「Claude デスクトップの画面では、MCP は **『コネクタ（Connectors / Capabilities）』** という名前で出てきます。呼び方が違うだけで同じものです。設定 → コネクタ / Capabilities から、使いたいサービスを追加して認証する——それが『MCP をつなぐ』操作です」。

   **③ MCP で何ができる？**
   「つないだサービスに対して、Claude が代わりに**探す・読む / 作る・書き込む / つなぐ・移す / まとめて任せる**を行えます。今回はこのうち、メールを**読む**“目”と、予定を**書く**“手”の両方を使います」。

3. そのうえで、このレベルでつなぐ MCP と完成イメージを1〜2文で示す。
   - 今日つなぐのは2つ:**メールを読む“目”（Gmail MCP）** と、**予定を書く“手”（Google カレンダー MCP）**。
   - 完成イメージ:「未読メールの中から日時のある予定を見つけて一覧化 → **あなたがOKしたものだけ**カレンダーに入る」。

> 補足: レベル4を先に体験した参加者には「あのとき使った PubMed も、同じ MCP（差込口）の仲間です」と一言添えると、つながりが伝わりやすい。

### Step 1. MCP の接続確認・設定（ここが今回の主役）

まず **Gmail MCP** と **Google カレンダー MCP** が使えるかを、実際に軽く呼び出して確認する。

1. **Gmail MCP** — 未読メールを1件だけ軽く取得して、接続できているかを確認する（件名が返れば OK）。
2. **Google カレンダー MCP** — カレンダー一覧（または直近の予定）を取得して、接続できているかを確認する（カレンダー名が返れば OK）。

結果を次の表で報告する（○＝利用可 / ×＝未接続）。

| 項目 | 状態 | メモ |
|---|---|---|
| Gmail MCP | | 接続アカウント |
| Google カレンダー MCP | | 既定カレンダー名 |

**未接続の場合の案内**（1〜2行で）:

> Claude デスクトップ → **設定 → コネクタ（Capabilities）** から **Gmail** と **Google カレンダー** を追加し、Google アカウントで OAuth 認証する。認証後、このチャットを再読み込みすると使えるようになる。

> 補足: 今回は `Gog-CLI` は使わない。`Gog-CLI` が入っている場合でも、本レベルでは Claude 標準の Gmail / カレンダー コネクタの方を使う、と明示する。

両方が ○ になったら Step 2 へ。片方でも × なら、まず接続を済ませてもらう（接続が完了するまで先へ進まない）。

### Step 2. 未読メールを読み取る

1. **未読メールのみ**を対象に取得する（例: `is:unread` 相当の条件）。件数が多い場合は直近のものから上限を決めて取得する（目安: 最大20〜30件）。
2. 取得したメールを、**差出人 / 件名 / 受信日時 / 要点1行**の一覧で参加者に見せる。
3. この時点では登録しない。「この中から“日時のある予定”を拾います」と伝える。

> 個人情報保護のため、本文をそのまま長々と画面に出さない。要点1行に要約して扱う。

### Step 3. 予定候補を抽出して整形

未読メールの中から、**日時が特定できる予定**（会議・面談・アポイント・学会・締切リマインド等）を抜き出し、カレンダー登録用に整形する。

各候補について、次を埋めた**候補テーブル**を提示する:

| # | 予定タイトル（案） | 日付 | 開始–終了 | 場所 / オンライン | 元メール（差出人・件名） | 確信度 |
|---|---|---|---|---|---|---|

整形ルール:
- **予定タイトル**は短く分かりやすく（例:「○○病院 症例検討会」）。機微情報は入れない。
- 終了時刻がメールに無ければ、デフォルト **1時間**として（推定である旨を明記）。
- 日付・時刻が曖昧（「来週」など特定不可）なものは**確信度: 低**として別枠にし、登録対象から外すか参加者に確認する。
- タイムゾーンは日本時間（JST）で扱う。

### Step 4. 重複チェック → 確認 → 登録

1. **重複チェック**: 各候補について、カレンダー MCP で同日・近い時間帯に既存予定がないかを確認する。重複の可能性があるものは「既に登録済みかも」と印を付ける。
2. **確認を取る**: `AskUserQuestion`（複数選択可）で「どれを登録しますか？」と提示し、参加者が選んだものだけを登録対象にする。**選ばれていないもの・確信度低のものは登録しない**。
3. **登録**: 選ばれた予定を、カレンダー MCP の予定作成で1件ずつ登録する。説明欄には**元メールの件名と差出人**程度（機微情報を除く）を残し、後から辿れるようにする。
4. 登録結果を表で報告する（登録済み / スキップ / 重複のためスキップ）。

### Step 5. うまくいったら Skill 化する

ここまでの流れが問題なく回ったら、**同じ手順を Skill（作業手順書）にまとめて再利用**できるようにする。

1. 1行で解説: 「毎回この指示を書かなくても、“スキル”にしておけば一言で同じ作業を呼び出せます」。
2. これまでの手順（未読取得 → 予定抽出 → 重複チェック → 確認 → 登録）を **SKILL.md** の形に整理する。最低限、次を含める:
   - いつ使うか（トリガー例: 「未読メールから予定を登録して」「メールチェックしてカレンダーに入れて」）
   - 使う MCP（Gmail / Google カレンダー）
   - 手順（本レベルの Step 2〜4）
   - 安全ルール（**確認してから登録**・**機微情報はタイトル/説明に入れない**・重複チェック）
3. 参加者の作業フォルダに `skill_mail-to-calendar/SKILL.md` として保存し、`present_files` で開けるようにする。`.skill` パッケージ化まで行える環境なら、それも案内する。
4. **発展（定期実行）**: 「毎朝この Skill を自動で走らせる」ことも可能だと伝える。スケジュール実行（毎朝○時に未読をチェックして候補を出す）と組み合わせると、いわゆる “メールチェックの自動化” になる。※自動実行時も**登録の最終確認**は残す設計が安全、と添える。

### Step 6. まとめ

「メールを読む“目”と予定を書く“手”を Claude につなぐと、“受信トレイ → 予定表”の転記がほぼ自動になります。さらに Skill 化＋定期実行で、毎朝の確認作業まで仕組み化できます」と締める。

> 安全の要点を一言: **登録は必ず確認してから**・**機微情報はカレンダーに残さない**・**重複は事前チェック**。

完了後「次はどのレベルを体験しますか？」と尋ね、`00_START` の Phase 1 に戻る。
