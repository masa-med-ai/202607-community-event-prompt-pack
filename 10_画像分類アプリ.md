### Step 0. 解説（つかみ）

1行で伝える: 「機械学習とは、正解付きのデータをたくさん見せて“見分け方”を自動で学ばせる技術です。今日はそれで『手書き数字を当てるアプリ』を作ります」。完成イメージ:「マスに数字を描くと、AIが 0〜9 のどれかを言い当てる。仕組みは胸部X線の正常/肺炎判定と同じ骨組み」。

### Step 1. データを見る（公開教材データ・ネット不要）

scikit-learn 同梱の **手書き数字データ（load_digits）** を読み込む。**8×8ピクセルの数字画像が約1,800枚**、ダウンロード不要。

```python
from sklearn.datasets import load_digits
d = load_digits()
print(d.images.shape, "枚 /", len(set(d.target)), "種類(0〜9)")
```

サンプル画像を数枚並べて表示（PNG → `present_files`）。「これは8×8＝64個の“明るさの数値”の集まり。AIはこの数値の並びから見分け方を学ぶ」と一言。

### Step 2. アルゴリズムの解説（★ここが核・必ず行う）

専門用語を避け、たとえ話で説明する。次の流れを、図やサンプルを見せながら**順に1つずつ**。

1. **画像＝数字の列**: 1枚の画像は「64個の数値（各マスの濃さ）」。AIにとって画像は“数の並び”でしかない。
2. **学習＝採点表づくり**: 今回のAI（**ロジスティック回帰**）がやるのは、**数字ごとに“採点表”を作ること**。「左上が濃いと＋」「真ん中が空いてると“0”っぽい」のように、**64マスそれぞれに点数（重み）**を割り当てる。最初はデタラメな採点表を、約1,800枚の正解付き画像で**少しずつ正解に近づくよう自動調整**する。これが「学習」。
3. **予測＝採点して一番高い数字を選ぶ**: 新しい画像が来たら、0用〜9用の10枚の採点表で**それぞれ合計点**を出し、**一番点が高い数字**を答えにする。確率（確信度）も出せる。
4. **なぜ訓練とテストを分ける？**: 覚えた問題で満点でも意味がない。**見せていない画像**で試して初めて実力（汎化性能）が分かる。だからデータを「学習用」と「テスト用」に分ける。
5. **精度と混同行列**: テストでの正答率（accuracy）が成績。**混同行列**を見ると「どの数字とどの数字を取り違えるか」というクセが分かる。

> ひとことまとめ:「学習＝たくさんの例から“数字ごとの採点表”を自動で作る」「予測＝採点して最高点を選ぶ」。深層学習（CNN）も、採点表が何層にも複雑になるだけで**骨組みは同じ**。

### Step 3. 学習させて評価する

データを学習用とテスト用に分け、**ロジスティック回帰**を学習 → テストで採点する。

```python
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score, confusion_matrix
Xtr,Xte,ytr,yte = train_test_split(d.data, d.target, test_size=0.25, random_state=0)
clf = LogisticRegression(max_iter=5000, C=0.05).fit(Xtr, ytr)
print("正答率:", round(accuracy_score(yte, clf.predict(Xte)), 3))   # 目安 0.96
```

- 正答率を伝える。混同行列を**ヒートマップ画像**にして `present_files`、「例えば 9 を 3 と少し取り違える、というAIのクセが見える」と Step 2-⑤ に接続。

### Step 3.5. 3つのやり方を比べる（ロジスティック回帰 / ランダムフォレスト / CNN）★

同じデータでも“見分け方”の作りが違うと結果が変わる。**3手法を学習させて正答率を並べる**。いずれも追加インストール不要・数秒で動く（CNN は NumPy だけで実装した小さなもの）。

- **ロジスティック回帰**: 画像を64個の数値に“ならして”、数字ごとの採点表で点数化（Step 2 の手法）。
- **ランダムフォレスト**: 「もし◯番目のマスが濃いなら…」という質問木（決定木）を大量に作り、多数決する。**画像を64個の数値の並びとして扱う**（2次元の形そのものは見ていない）。
- **CNN（畳み込みニューラルネット＝深層学習）**: **小さなフィルタ（3×3）を画像の上で滑らせて、エッジ・曲線などの“局所的な形”を検出**する。flatten せず**2次元の構造をそのまま活かす**のが本質。複雑な画像ほど有利になる。

進め方:
1. 3手法の正答率を**棒グラフ**にして `present_files`（目安: LR 0.93 / RF 0.98 / CNN 0.96 前後。**この小さな8×8データでは RF も強い**——「CNNが常に最強ではない。データと規模次第」と伝えるのが大事）。
2. **CNN が学習した3×3フィルタを画像**にして見せる（`present_files`）。「この小さな模様が“エッジ検出器”のように働き、画像の中の形に反応する。これが“CNNは形を見る”の正体」と説明する。
3. ひとこと:「**ランダムフォレスト＝画像を数値の並びとして賢く分類**／**CNN＝画像の2次元の形を直接とらえる**。大きく複雑な画像（X線・病理など）になるほど CNN の強みが出る」。

> facilitator 用メモ: CNN は `conv(3×3,8)→ReLU→flatten→dense(10)→softmax` の最小構成を NumPy の行列演算（im2col）で実装し、ミニバッチSGDで学習すれば 8×8 データで数秒・正答率0.95前後。**同梱の `10_画像分類アプリ_サンプル出力/`** に比較棒グラフ `model_comparison.png` と学習フィルタ `cnn_filters.png` の完成例がある。本格的な CNN（28×28以上・PyTorch/Keras）は末尾の Code 版付録で。

### Step 4. アプリにする（★作って動かす体験）

学習しただけでは“箱の中”。これを**ブラウザで触れるアプリ**にする。

1. 学習済みモデルの**採点表（重み `clf.coef_` と `clf.intercept_`）を取り出して JSON 化**し、**1枚の自己完結 HTML に埋め込む**（外部送信なし・完全ローカル）。
2. アプリの中身:
   - **描画キャンバス**（マウス/指で数字を描く）→ 8×8 に縮小 → 選んだエンジンで計算 → **予測した数字**と**各数字の確信度バー**を表示。
   - **エンジン切替トグル（ロジスティック回帰 ⇄ CNN）**: 同じ入力を2つのエンジンで予測し、結果や確信度の違いを比べられる（Step 3.5 の体感をアプリ上で再現）。
   - 「**学習データの例**」サムネイルも置き、クリックすると“AIが得意な入力”での動作を確認できる（描いた絵がうまく当たらない時の対比になる）。
3. `digit_app.html` として保存し、`present_files` で開く。**参加者に実際に数字を描いて、エンジンを切り替えて試してもらう**。
4. **気づきを引き出す**: 自分の手書きは外すこともある。「学習データ（きれいに整形された数字）と、自分の描き方が違うから」。**“データの偏り”が精度に効く**という、実務でも重要な教訓につなげる。

> 生成のコツ（facilitator 用）: **LR** は重み `coef_`/`intercept_` を JSON 化し、JS で `logits[k]=Σ coef[k][j]*x[j]+intercept[k]` → softmax。**CNN** は `Wc(9×8)/bc/Wd(288×10)/bd` を JSON 化し、JS で `conv(3×3) → ReLU → flatten → dense → softmax` を再現（LRは入力0〜16、CNNは0〜1に正規化する点に注意）。キャンバスは 8×8 に縮小し最大濃度を16に正規化。**同梱のサンプル `10_画像分類アプリ_サンプル出力/digit_app.html` がLR/CNN切替式の完成例**（JS推論がPython側と一致することを検証済み）。

```python
from medmnist import PneumoniaMNIST
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

train = PneumoniaMNIST(split="train", download=True, size=28)
test  = PneumoniaMNIST(split="test",  download=True, size=28)
Xtr = train.imgs.reshape(len(train.imgs), -1); ytr = train.labels.ravel()
Xte = test.imgs.reshape(len(test.imgs), -1);  yte = test.labels.ravel()
clf = LogisticRegression(max_iter=2000).fit(Xtr, ytr)
print("正答率:", round(accuracy_score(yte, clf.predict(Xte)), 3))
```
