# FAQ — つまずいたら（参加者・ファシリテーター共通）

> ワークショップ中によくあるトラブルと対処のまとめ。Claude への指示: 参加者がエラーや接続不良で止まったら、このファイルの該当項目を基に**1〜2行で**対処を案内し、解決したら元の Step に戻ること。

---

## MCP・接続まわり

**Q1. PubMed MCP が見つからない／`count` がエラーになる（Level 4・5・9）**
- 作業フォルダの `MCP/pubmed-mcp-bundle.mcpb` から導入する（`00_START` Phase 0 の手順参照）。
- **Windows**: 自動導入不可。`images/00_MCPBをインストールする方法.png` を見ながら、`.mcpb` を Claude Desktop の設定画面へドラッグ＆ドロップする。
- 導入後は **Claude Desktop を再読み込み**（または再起動）しないと認識されない。

**Q2. Gmail / Google カレンダーのコネクタが無い（Level 8・9）**
- Claude デスクトップ → 設定 → コネクタ から追加し、OAuth 認証する（Claude 標準コネクタを使う。当日その場でも接続可能）。
- 認証画面が出ない場合は、既定ブラウザで Google にログインしているか確認する。

**Q3. MCP を追加したのに Claude が認識しない**
- 接続系の変更（MCP・コネクタ・スキル）は**新しいチャット／再読み込み後**に反映されることが多い。まず Claude Desktop を再読み込みする。

## スキルまわり

**Q4. `pubmed-systematic-review` などのスキルが見えない（Level 5）**
- 本フォルダ同梱の `Agent-Skills-main/pubmed-systematic-review.skill` から導入できる。導入できない場合は、`.skill` を展開して中の `SKILL.md` を Claude に読ませて手動進行も可能。

## サンドボックス・ネットワーク

**Q5. Cowork の中からダウンロードやサイト閲覧ができない（Level 11 ほか）**
- Cowork の実行環境（サンドボックス）は**ネットワーク制限あり**。Level 11 の本編がオフラインの教材データを使うのはこのため。MedMNIST など要ダウンロードの発展課題は **Desktop 版 Code / CLI** で行う。

## 定期実行タスク

**Q6. 登録した定期タスクが動かない（Level 9・12）**
- 定期タスクは **PC の電源が ON ＋ Cowork（Claude Desktop）が起動中**でないと実行されない。まず「今すぐ試走」で中身を確認し、当日は試走の成功をもってゴールとしてよい。

## Word / ファイルまわり

**Q7. Word テンプレートの差し込みでレイアウトが崩れた・エラーになる（Level 3）**
- 原本 `理由書フォーマット.docx` は**上書きせず必ず別名保存**。崩れたら別名ファイルを削除して原本からやり直せばよい。
- このテンプレは unpack→pack 検証でまれにエラーになるため、`word/document.xml` を直接置換して zip を書き戻す方式が安全（詳細は README 参照）。

**Q8. 作ったファイルが見当たらない**
- 成果物は作業フォルダ（このフォルダ）に保存される。チャット内のファイルカードをクリックすると開ける。見つからなければ「さっき作ったファイルを開いて」と Claude に頼めばよい。

## Claude Code（Level 10）

**Q9. 「Code」タブが無い／使えない**
- Claude Desktop に**サインイン済み**で、**Pro/Max/Team/Enterprise** プランであることが前提。Windows は初回に Git for Windows の導入を求められることがある。

**Q10. 実ファイルが書き換わるのが不安**
- 最初は **Ask permissions（確認）モード**のまま使い、差分を読んでから承認する。秘密情報・患者情報は渡さない。

## 進行まわり

**Q11. 途中で別の Level に切り替えたい**
- 「別の Level にしたい」と伝えれば、`00_START` の Phase 1（レベル選択）に戻れる。途中まで作った成果物はフォルダに残る。

**Q12. Claude が先に進みすぎる／止まってしまった**
- 「1 Step ずつ、確認しながら進めて」と伝え直せば `CLAUDE.md` の進行ルールに復帰する。止まったままなら「次の Step に進んで」と合図する。
