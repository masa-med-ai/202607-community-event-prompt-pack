1

MCP は「共通の差込口」

スマホやノートPCの USB-C のように、規格が共通だから、いろいろなサービスを Claude にそのまま“差し込める”。

要点：MCP に対応したサービスは Claude に“差し込む”だけ。Claude が自分でそのサービスを操作できるようになる ＝ 「会話だけ」から「道具を実際に使える」へ。
2

MCP ＝「コネクタ」

Claude デスクトップの画面では、MCP は 「コネクタ（Connectors / Capabilities）」 という名前で出てくる。呼び方が違うだけで同じもの。
MCP ＝ コネクタ ＝ 設定 → Capabilities から追加する“差込口”
迷ったら：設定 → コネクタ / Capabilities を開き、使いたいサービス（Gmail・カレンダーなど）を追加して OAuth 認証する。それが「MCP をつなぐ」操作。
3

MCP でできること

つないだサービスに対して、Claude が代わりに“操作”する。代表的な4つ。
🔎
探す・読む
未読メールや論文、予定を検索して中身を取得する。
✍️
作る・書き込む
カレンダーに予定を登録、下書きメールを作成する。
🔄
つなぐ・移す
メールの情報を予定表へ、論文を執筆ツールへ橋渡しする。
🤖
まとめて任せる
複数サービスをまたぐ作業を一連の流れとして実行する。
＝ Claude が“目（読む）”と“手（書く）”を手に入れる。今回はその両方を使う。
7

このレベルでつなぐ MCP

未読メールから予定を拾って、確認のうえカレンダーに登録する。使うのは2つの MCP。
👀
読む“目”
Gmail MCP
未読メールを読み、日時のある予定を見つける。
✋
書く“手”
Google カレンダー MCP
あなたが OK したものだけ、予定として登録する。
完成イメージ：未読メール → 日時のある予定を一覧化 → あなたが選んだものだけ → カレンダーに登録
安全の要点：登録は必ず確認してから・機微情報はタイトル/説明に残さない・重複は事前チェック。
